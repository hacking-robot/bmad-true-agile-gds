# Sprint Review Checklist

Use this checklist to ensure sprint review is complete.

## ✅ Review Complete When:

### Story Status Verified
- [ ] All stories accounted for (done, carryover, or returned to backlog)
- [ ] Done stories have acceptance criteria verified
- [ ] Incomplete stories have resolution documented
- [ ] Carryover stories are clearly identified

### Velocity Calculated
- [ ] Sprint velocity calculated (sum of completed story points)
- [ ] Velocity log updated with sprint entry
- [ ] Averages recalculated (last 3 sprints, all-time)
- [ ] Trend determined (improving/stable/declining)

### Sprint File Updated
- [ ] Status changed to `completed`
- [ ] Actual end date recorded
- [ ] Completed points recorded
- [ ] Carryover stories listed
- [ ] Review file reference added

### Review Document Created
- [ ] Sprint review summary document created
- [ ] Contains sprint metrics
- [ ] Contains incomplete story resolutions
- [ ] Contains velocity analysis

---

## Incomplete Story Resolution Options

| Option | When to Use | Effect |
|--------|-------------|--------|
| **Carryover** | Story was in-progress, close to done | Automatically added to next sprint |
| **Return to Backlog** | Story needs re-prioritization | Goes back to epic, can be reselected later |
| **Split** | Story too large | Requires backlog-refinement to break into smaller stories |

---

## Velocity Health Indicators

| Velocity % | Health | Action |
|------------|--------|--------|
| > 90% | Excellent | Maintain current approach |
| 70-90% | Good | Minor adjustments |
| 50-70% | Concerning | Review in retrospective |
| < 50% | Problem | Investigate blockers, consider capacity reduction |
